# Features

Detailed documentation of every feature in PlayForge, organized by category with implementation file paths.

## Play Designer

### Canvas Engine
**What:** A Konva.js-based interactive football field where coaches draw plays.
**How:** The canvas renders at a logical 1000x600 resolution and scales responsively to the container. Three layers handle the field background, coverage overlays, and interactive elements (players + routes).

| File | Purpose |
|------|---------|
| `src/engine/play-canvas.tsx` | Main canvas component with layer management, pointer handling, and drawing state |
| `src/engine/field-renderer.tsx` | Grass gradient, yard lines, hash marks, end zones with text, vignette overlay |
| `src/engine/constants.ts` | Field dimensions, colors, player radius, hash mark ratios |

### Route Drawing
**What:** Click-to-place waypoint system for drawing player routes on the canvas.
**How:** Activate drawing mode (`D` key or toolbar toggle). Click a player to start their route, click the field to add waypoints, and double-click, press Enter, or click the same player to finish. Right-click removes the last waypoint. A dashed preview line follows the cursor from the last waypoint.

When a route is finished, `detectRouteType()` analyzes the waypoint geometry to auto-label the route (Go, Slant, Post, Corner, Curl, Dig, Out, Flat, Drag).

| File | Purpose |
|------|---------|
| `src/engine/play-canvas.tsx` | Drawing state machine, waypoint placement, preview line |
| `src/engine/constants.ts` | `detectRouteType()` — classification by angle, depth, and horizontal movement |
| `src/engine/route-line.tsx` | Route rendering with arrowheads, waypoint dots, and line styles |
| `src/engine/types.ts` | `Route`, `RouteWaypoint` type definitions |

### Route Templates
**What:** 40+ predefined route templates across 5 categories that can be applied to any player with one click.
**How:** Each template defines waypoint offsets relative to a player's position. `applyRouteTemplate()` converts these to absolute coordinates. The route picker panel (`R` key) displays templates grouped by category.

**Categories:**
- **Short** (10): Slant, Quick Out, Hitch, Flat, Drag, Whip, Bubble Screen, Stick, Speed Out, Quick In
- **Medium** (10): Dig/In, Out, Curl/Comeback, Over, Sail, China, Angle, Texas, Wheel (Short), Option
- **Deep** (10): Go/Fly, Post, Corner, Post-Corner, Corner-Post, Fade, Deep Out, Deep In, Seam, Skinny Post
- **Screen** (3): RB Screen, WR Screen, Tunnel Screen
- **Block** (7): Pass Protection, Pull Block, Drive Block, Kick-Out Block, Chip & Release, Lead Block, Zone Block

| File | Purpose |
|------|---------|
| `src/engine/routes-library.ts` | Route template definitions, `applyRouteTemplate()`, category helpers |
| `src/components/play/route-picker.tsx` | UI panel for browsing and applying route templates |

### Formations
**What:** 20+ pre-built formation templates for offense and defense across all game formats.
**How:** Each formation defines player positions (id, label, x, y, side). Selecting a formation replaces all canvas players and clears routes/motions.

**Offense:** Shotgun 2x2, I-Formation, Singleback, Pistol, Empty, Shotgun Trips, plus flag football variants (4v4 Spread/Diamond, 5v5 Spread/Stack/Trips, 6v6 Spread/Trips, 7v7 Spread/Trips/Empty/Bunch, 8v8 Spread, 9v9 Spread)

**Defense:** 4-3, 3-4, Nickel, Dime

| File | Purpose |
|------|---------|
| `src/engine/constants.ts` | `FORMATIONS` array, `getFormationById()`, `getFormationsBySide()`, `getFormationsByFormat()` |
| `src/components/play/formation-picker.tsx` | Formation selection UI with side/format filtering |

### Play Animation
**What:** Animated playback showing players running their routes with ball flight and QB read progression.
**How:** `generateKeyframes()` samples 30 positions along each player's route waypoints using cumulative distance interpolation. `getAnimationFrame()` computes all positions at a given time, including parabolic ball flight (for pass plays, thrown at t=0.6) and sequential QB read indicators.

| File | Purpose |
|------|---------|
| `src/engine/animation-engine.ts` | Keyframe generation, time interpolation, ball flight, read order |
| `src/components/play/animation-controls.tsx` | Play/pause/speed controls, timeline scrubber |
| `src/engine/ball.tsx` | Football sprite component |
| `src/engine/read-indicator.tsx` | Numbered read progression bubbles |

### AI Play Generation
**What:** Describe a play in natural language and receive a complete diagram generated by Claude AI.
**How:** The client sends a description to `POST /api/ai/generate-play`. The server constructs a detailed system prompt containing the coordinate system, all formations, and all route templates, then sends to Claude claude-sonnet-4-20250514. The response is parsed as `CanvasData` JSON and loaded into the designer.

**Prerequisite:** Set `ANTHROPIC_API_KEY` in `.env`.

| File | Purpose |
|------|---------|
| `src/lib/ai/play-generator.ts` | System prompt construction, API call, response parsing |
| `src/app/api/ai/generate-play/route.ts` | REST endpoint with auth and error handling |
| `src/components/play/ai-generator.tsx` | Client panel with description input and generation controls |

### Play Mirroring
**What:** Horizontally flip any play across the field center (x=500).
**How:** `mirrorPlay()` transforms all player x-coordinates, route waypoints, and motion paths: `newX = 500 + (500 - oldX)`. In the designer, `H` key mirrors in-place. The `mirrorPlayAction()` server action creates a new database record named "(Mirrored)".

| File | Purpose |
|------|---------|
| `src/engine/mirror.ts` | `mirrorPlay()` — pure function that mirrors `CanvasData` |
| `src/lib/actions/play-actions.ts` | `mirrorPlayAction()` — creates mirrored copy in database |

### Coverage Overlays
**What:** Visual zone coverage diagrams overlaid on the field showing defensive responsibilities.
**How:** 5 coverage schemes (Cover 0, 1, 2, 3, 4) defined as arrays of colored rectangles with labels. Rendered as a non-interactive Konva layer between the field and players.

**Schemes:** Cover 0 (Blitz, all man), Cover 1 (Man Free), Cover 2 (2 deep halves, 5 under), Cover 3 (3 deep thirds, 4 under), Cover 4 Quarters (4 deep, 3 under).

| File | Purpose |
|------|---------|
| `src/engine/coverage-zone.tsx` | `COVERAGE_SCHEMES`, `CoverageOverlay` component, `getCoverageScheme()` |

### Pre-Snap Motion
**What:** Diagram player motion before the snap with animated arrows.
**How:** Toggle motion mode (`M` key), click a player to select them, click a field position to set the destination. Creates a `MotionPath` record and moves the player to the new position. The motion arrow renders as a curved dashed line from the original to final position.

| File | Purpose |
|------|---------|
| `src/engine/motion-arrow.tsx` | MotionArrow rendering component |
| `src/engine/play-canvas.tsx` | Motion mode interaction handling |
| `src/engine/types.ts` | `MotionPath` type definition |

### Export to PNG
**What:** Download the current play as a high-resolution PNG image.
**How:** Uses Konva's `stage.toDataURL()` at 2x pixel ratio, creates a temporary download link, and triggers the download. Filename is sanitized from the play name.

| File | Purpose |
|------|---------|
| `src/engine/export.ts` | `exportPlayAsImage()` function |

### Print Layout
**What:** Print-optimized views for playbook sheets and wristband cards.
**How:** Two modes: "playbook" (full-page play diagrams) and "wristband" (compact card grid). Accessed via the printer icon in the toolbar.

| File | Purpose |

### Version History
**What:** Every save creates an automatic snapshot. Previous versions can be browsed and restored.
**How:** `updatePlay()` creates a `PlayVersion` record with the current canvas/animation data before writing the new data. `restorePlayVersion()` snapshots the current state as a safety net, then overwrites with the selected version's data.

| File | Purpose |
|------|---------|
| `src/components/play/version-history.tsx` | Version list UI with restore button |
| `src/lib/actions/play-actions.ts` | `getPlayVersions()`, `restorePlayVersion()` |

### Play Library
**What:** 30+ pre-built play templates organized by category for quick starting points.
**How:** Templates define a formation ID and route assignments per player position. Loading a template picks the formation, places players, and applies routes from the route library.

**Categories:** Quick Game (5), Play Action (4), Deep Shots (4), Run Game (5), Screens (3), Red Zone (4)

| File | Purpose |
|------|---------|

| `src/components/play/play-library.tsx` | Library browser panel |

### Film Link
**What:** Attach a film URL and timestamp to a play for quick reference.
**How:** Stores `filmUrl` and `filmTimestamp` on the `Play` record. The `appendTimestampToUrl()` utility appends `?t=<seconds>` to the URL.

## Learning System

### Spaced Repetition
**What:** SM-2 algorithm schedules play reviews based on demonstrated mastery.
**How:** Each play view and quiz score updates the player's ease factor and review interval. Plays with quality < 3 (score < 60%) reset to a 1-day interval. Successfully recalled plays get progressively longer intervals (1 day -> 3 days -> interval * easeFactor).

| File | Purpose |
|------|---------|
| `src/lib/spaced-repetition/sm2.ts` | `calculateNextReview()`, `qualityFromScore()`, `masteryFromInterval()` |
| `src/lib/actions/progress-actions.ts` | `recordPlayView()`, `recordQuizScore()` — update SM-2 state in database |

### Quizzes
**What:** Assessment system with 5 question types linked to plays.
**How:** Coaches create quizzes (optionally tied to a game plan) and add questions per play. Players complete quizzes, and their per-play progress is updated based on question accuracy.

**Question Types:**
- `multiple_choice` — Select one correct answer from options
- `tap_field` — Tap a zone on the field (correctZone JSON)
- `identify_route` — Name the route shown
- `situation_match` — Match situations to appropriate plays
- `assignment_recall` — Recall your assignment for a given play

| File | Purpose |
|------|---------|
| `src/lib/actions/quiz-actions.ts` | CRUD for quizzes, questions, and attempt submission |
| `src/components/quiz/quiz-flow.tsx` | Quiz-taking flow component |
| `src/components/quiz/multiple-choice.tsx` | Multiple choice question renderer |
| `src/components/quiz/quiz-card.tsx` | Quiz list item card |
| `src/app/(coach)/quizzes/create/` | Quiz creation wizard |
| `src/app/(player)/quiz/[id]/page.tsx` | Player quiz-taking page |

### Player-Specific View
**What:** Players see only plays from the active game plan with their mastery status.
**How:** The player layout fetches the active game plan for their org and displays assigned plays. Each play shows mastery level, view count, and next review date.

| File | Purpose |
|------|---------|
| `src/app/(player)/plays/page.tsx` | Player play list filtered by game plan |
| `src/app/(player)/plays/[id]/page.tsx` | Individual play view with assignment toggle |
| `src/components/play/player-plays-filters.tsx` | Mastery and formation filters for player view |

### Gamification
**What:** XP, levels (Rookie through MVP), and 10 earnable badges motivate players to study.
**How:** XP is calculated from views (10), quizzes (50), mastery (100), and streaks (25). Levels unlock at XP thresholds. Badges are earned for milestones like first view, perfect quiz score, 7-day streak, and mastering all plays.

| File | Purpose |
|------|---------|
| `src/lib/gamification.ts` | `calculateXP()`, `getLevel()`, `getEarnedBadges()`, `BADGES` definitions |
| `src/components/player/player-home-client.tsx` | Player home with XP bar, badges, and level display |

## Coach Tools

### Analytics Dashboard
**What:** Team-wide metrics including install completion, quiz averages, mastery heatmaps, and inactive player detection.
**How:** `getTeamAnalytics()` aggregates data across memberships, player progress, and quiz attempts. Install completion is the percentage of players who have viewed all active game plan plays. Inactive players are those with no activity in 3+ days.

| File | Purpose |
|------|---------|
| `src/lib/actions/analytics-actions.ts` | `getTeamAnalytics()`, `getInstallProgress()`, `getLeaderboard()`, `getPlayerRank()` |
| `src/app/(coach)/analytics/page.tsx` | Analytics page |
| `src/components/analytics/stat-card.tsx` | Individual metric card |
| `src/components/analytics/mastery-heatmap.tsx` | Player-by-play mastery grid |
| `src/components/analytics/leaderboard.tsx` | Ranked player table with composite scores |
| `src/components/analytics/install-tracker.tsx` | Per-play install progress bars |

### Game Plans
**What:** Weekly game plans that organize plays in a specific order with opponent and situation context.
**How:** Each org can have multiple game plans, but only one is "active" at a time. The active plan determines which plays appear in the player view. Plays can be added, removed, and reordered within a plan.

| File | Purpose |
|------|---------|
| `src/lib/actions/game-plan-actions.ts` | CRUD + reorder + active plan management |
| `src/app/(coach)/game-plans/page.tsx` | Game plan list |
| `src/app/(coach)/game-plans/[id]/page.tsx` | Game plan detail with ordered play list |
| `src/components/game-plan/play-list.tsx` | Sortable play list component |

### Practice Plans
**What:** Practice schedules with named periods, durations, play assignments, and notes.
**How:** A practice plan contains ordered periods. Each period has a name, duration in minutes, optional play IDs, and notes. Periods can be reordered and edited inline.

| File | Purpose |
|------|---------|

### Roster Management
**What:** View and manage team members with role and position assignment.
**How:** Coaches see all org members with their role, position, and recent activity. Owners can change roles. Coaches and owners can remove members. Invite codes can be regenerated.

| File | Purpose |

### Playbook Sharing
**What:** Share playbooks with other organizations and import shared playbooks.
**How:** `sharePlaybook()` creates a `PlaybookShare` record linking the playbook to the target org (found by slug or invite code). `importSharedPlaybook()` deep-copies the playbook and all its plays into the receiving org. Shares can be revoked at any time.

| File | Purpose |
|------|---------|
| `src/lib/actions/playbook-actions.ts` | `sharePlaybook()`, `getSharedPlaybooks()`, `revokePlaybookShare()`, `importSharedPlaybook()` |
| `src/components/play/share-playbook.tsx` | Share dialog with org slug input |
| `src/app/(coach)/playbooks/import-button.tsx` | Import button for shared playbooks |

### Notifications
**What:** Context-aware alerts for coaches (inactive players, low scores, install progress) and players (due reviews, pending quizzes, new plays).
**How:** `generateCoachNotifications()` and `generatePlayerNotifications()` create notification objects from analytics and progress data. Each notification has a type, message, timestamp, and link.

| File | Purpose |
|------|---------|

### QR Code Invites
**What:** Generate QR codes that link to the team join page with the invite code pre-filled.
**How:** `generateInviteQR()` uses the `qrcode` library to create a data URL encoding the join URL with the code parameter.
